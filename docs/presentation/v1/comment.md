## 1p — 타이틀 (약 10초)

안녕하세요. AI 여행 동행자 **Polylog**입니다.
여행에서 마주치는 귀찮은 일 — 외국어 리뷰 읽기, 지나친 명소, 새는 지출, 모르고 주문한 메뉴 —
이 모든 잡일을 **AI가 대신**하고, 사용자는 여행에만 집중하게 만드는 앱입니다.

## 2p — 문제 정의 (약 15초)

저희는 여행자가 겪는 **네 가지 불편**에서 출발했습니다.

이 네 가지를 각각 하나의 AI 기능으로 해결합니다.

## 4p — 기능: 어떻게 동작하는가 (약 30초) ★

네 기능은 모두 **"입력 → Lambda → AWS AI/외부 API → 저장 → 결과"** 라는 동일한 파이프라인을 따릅니다.

**① 메인 — AI 장소 추천**
GPS 좌표와 카테고리를 받으면, `fn-recommend`가 **Google Places API**로 반경 내 후보(이름·별점·리뷰)를 수집하고,
그 목록을 **Bedrock의 Claude**에 넘깁니다. Claude는 단순 나열이 아니라 *"이 라멘집은 점심에 대기가 짧고 합리적"* 처럼
**원어민 리뷰를 분석한 개인화 추천과 이유**를 생성합니다. 광고성 리뷰는 의심 표기합니다.

**② 메뉴판 번역**
사진을 S3에 올리면 `fn-menu`가 **Textract(DetectDocumentText)** 로 메뉴 텍스트를 뽑고,
**Translate**로 번역한 뒤, **Claude**가 사용자 알레르기·식이 제한을 반영해 추천 메뉴를 골라줍니다.

**③ 영수증 가계부**
`fn-receipt`가 **Textract(AnalyzeExpense)** 로 금액·항목·날짜·통화를 구조화 추출하고,
**환율 API**로 원화 환산, **Claude**가 카테고리를 자동 분류해 가계부에 기록합니다.

**④ AI 일정**
`fn-schedule`이 **Places(Text Search)** 로 장소를 찾고, **이전 대화 이력과 기존 일정을 함께** Claude에 넣어
대화형으로 일정을 추천·수정합니다.

> 핵심: **AWS AI 3종(Bedrock·Textract·Translate)을 한 파이프라인에 유기적으로 결합**한 것이 차별점입니다.

## 5p — 설계·아키텍처: 왜 이 기술인가 (약 35초) ★★

전체 구조는 **완전 서버리스**입니다. 화면 흐름을 따라가며 기술 선택 이유를 말씀드리면 —

- **Flutter** : Android·iOS를 한 코드베이스로 — PoC 일정(4주)과 인력 대비 가장 효율적이라 선택했습니다.
- **API Gateway + Lambda** : 상시 가동 서버가 **0대**입니다. 요청당 과금이라 트래픽이 적은 PoC에서
  **무료 티어 내 월 2~3달러**로 운영됩니다. 서버 관리 부담도 없습니다.
- **소셜 OAuth(Google) + `fn-authorizer`** : 관리자 IAM 제약으로 Cognito를 쓸 수 없어,
  Google 로그인으로 받은 **ID 토큰을 Lambda Authorizer가 JWKS로 검증**하는 무상태 인증을 택했습니다.
  덕분에 별도 인증 서버 없이 안전하게 인가합니다.
- **Bedrock(Claude)** : 추천·번역 후처리·분류·대화까지 **자연어 추론이 필요한 모든 지점**에 공통으로 씁니다.
  지연을 줄이려 **가벼운 Haiku 모델**과 **프롬프트 토큰 최소화**를 적용했습니다.
- **Textract / Translate** : OCR과 번역은 검증된 관리형 AI 서비스를 써서 정확도와 개발 속도를 확보했습니다.
- **DynamoDB** : `trip_id`를 파티션 키, **시각을 정렬 키**로 두어 일정·지출이 *자동으로 시간순 정렬*됩니다.
  On-Demand 용량이라 사용한 만큼만 과금됩니다.
- **S3** : 사진·영수증은 **SSE 암호화**로 저장하고, CloudFront 대신 **Presigned URL**로 직접 업로드합니다.

> 한마디로 **"관리형 서비스로 운영 부담은 0, AI 추론은 Bedrock으로 일원화"** 한 설계입니다.

### 부록 — 예상 질문 대비 (Q&A)

- **왜 Cognito가 아닌 소셜 OAuth?** 관리자 IAM에서 Cognito 권한이 제한돼, Google ID 토큰을 `fn-authorizer`가 JWKS로 검증하는 무상태 방식으로 우회.
- **Bedrock 리전?** 한국어 처리는 서울, Bedrock은 us-east-1 cross-region 호출(~200ms). 프롬프트 최적화로 상쇄.
- **왜 DynamoDB(NoSQL)?** 여행 단위 단순 액세스 패턴 + On-Demand 과금이 PoC 규모에 적합. 단일 테이블 설계로 타임라인 정렬을 키 설계로 해결.
- **월 비용?** 무료 티어 적용 시 약 $2~3, 종료 후 $10~15 수준.
