## 0. 핵심 전략 — 왜 "묶음"인가

채점표는 1점 단위 33항목(Cat1 10 + Cat2 5 + Cat3 10 + Cat4 5 + 과제 15·문서 5 + 가산점)이지만,
**Cat 1·2·3이 "적용기술·아키텍처·개발환경·빌드배포"를 반복 채점**한다. 따라서 5분 안에 모든 항목을 따로 말할 필요가 없다 —
**한 번 말한 내용이 여러 항목에 동시 득점되도록** 6개 그룹으로 묶는다.

| 그룹 | 한 줄 요지 | 동시 커버 |
|---|---|---|
| A 오프닝 | 비전 + 4대 불편 | Cat1 ×4 |
| B 아키텍처 ★ | 다이어그램 1장 + "왜 이 기술" + 최적화 | ~9 |
| C 프로세스 | WBS·배포·테스트·문서·잔디 1장 | ~9 |

| E 데모 | 30초 시나리오 = 활용방안 | Cat4 ×5 |
| F 마무리 | 향후 + 나만의 기법(가산점) | Cat1 + 가산점 |

## 2. 구간별 상세 설계

### [1] 비전·문제정의 — 0:40  (그룹 A)
**동시 득점**: Cat1 `비전 표현`·`비전 와닿음`·`문제정의 표현`·`문제정의 공감대`
**말할 핵심**
- 한 문장 비전: "여행의 잡일(외국어 리뷰·지나친 명소·새는 지출·모르는 메뉴)을 **AI가 대신**하고 사용자는 여행에만 집중."
- 4대 불편 → 각각 하나의 AI 기능으로 1:1 해결한다고 선언 (= 공감대 + 구조 예고)
**슬라이드**: 1p 타이틀 / 2p 4대 불편 아이콘
**주의**: 대사 암기·연습 필수 (Cat1 `대사 준비 연습` 항목 — 읽으면 감점).

### [2] 4기능 키워드 — 0:20
**동시 득점**: Cat1 `어떻게 구현했는지` 도입
**말할 핵심**: "맞춤 장소 추천 · 대화형 AI 일정 · 메뉴판 번역 · 영수증 가계부" 4키워드 압축, "지금부터 동작과 기술" 예고.

### [3] 데모 30초 — 0:30  (그룹 E)
**동시 득점**: Cat4 `30초 데모`·`앱 형태`·`기본 기능`·`순차 매끄러움`·`임팩트` + Cat1 `활용방안`
**시나리오(순차)**: 로그인된 앱 → 위치 기반 **장소 추천 카드**(AI 추천 이유 노출) → 탭 전환으로 **메뉴판 촬영 → 번역+추천** 1컷 → **영수증 → 자동 가계부** 1컷.
**임팩트 포인트**: "AI 추천 이유" 텍스트가 화면에 뜨는 순간 / 외국어→한국어 번역 전환 순간.
**준비**: 네트워크·계정·더미 데이터 사전 세팅, 실패 대비 녹화 영상 백업.

### [4] 아키텍처·적용기술·성능 — 1:10  (그룹 B ★ 최고 효율)
**동시 득점(약 9항목)**:
- Cat1 `적용기술 표현`·`어떻게 구현했는지`
- Cat3 `적용기술 (읽지않고) 설명`·`아키텍처 (이해하고) 설명`·`성능 최적화`
- Cat2 `앱 구조(레이어/디렉토리)`·`ADR 근거`
**말할 핵심 — 화면 흐름 따라 "왜 이 기술"**
- **공통 파이프라인**: 입력 → API Gateway → Lambda → (AWS AI/외부 API) → DynamoDB/S3 → 결과. 4기능 모두 동일 패턴.
- **Flutter**: 한 코드베이스, 4주 PoC에 최효율 (ADR-003)
- **API Gateway + Lambda(6종)**: 상시 서버 0대, 요청당 과금 → **무료티어 월 $2~3** (ADR-001)
- **소셜 OAuth(Google) + fn-authorizer**: ID 토큰을 tokeninfo로 검증하는 **무상태 인가** (ADR-007)
- **Bedrock(Claude)**: 추천·번역후처리·분류·대화 — 자연어 추론 전 지점 일원화 (ADR-004)
- **Textract/Translate**: 검증된 관리형 OCR·번역으로 정확도+속도 (ADR-005)
- **DynamoDB**: `trip_id` 파티션 + 시각 SK → 일정·지출 **자동 시간순 정렬** (ADR-014/015)
- **S3**: SSE 암호화 + Presigned URL (CloudFront 미사용, ADR-008)
**★ 성능 최적화 한 줄씩 끼워넣기(= Cat3 성능 항목 득점)**:
"지연을 줄이려 **Haiku 모델 + 프롬프트 토큰 최소화**", "정렬은 **키 설계로 비용 0**", "서버 0대로 **비용도 최적화**".
**앱 구조(= Cat2)**: "Flutter는 feature 단위 디렉토리(`features/recommend·menu·receipt·schedule`), 백엔드는 기능별 Lambda 분리" 한 문장.
**슬라이드**: 4p 4기능 파이프라인 / 5p 전체 아키텍처 다이어그램 (이 한 장이 핵심).

### [5] ADR 시행착오 — 0:50  (그룹 D ★)
**동시 득점**: Cat3 `구현 시행착오(개발자 관점)`·`지속적 개선/습득 의지` + Cat2 `ADR 최소 3개`·`ADR 기준 정확 답변`
**말할 핵심 — "결정을 번복한 4가지" (개발자 스토리)**
1. **Cognito → 소셜 OAuth 우회** (ADR-007): 관리자 IAM에서 Cognito 차단 → ID 토큰 직접 검증으로 전환. *제약을 만나 설계를 바꾼 사례.*
2. **CloudFront 철회** (ADR-008): PoC 트래픽엔 CDN 과투자 + 영수증은 개인정보 → Presigned URL. *오버엔지니어링 회피.*
3. **AWS AI 7종 → 3종 깊이우선** (ADR-005): 넓게보다 깊게 — Bedrock 4패턴·Textract 2 API. *범위 재조정.*
4. **테이블 trip_id 통합** (ADR-014/015): standalone UUID PK가 Scan 유발 → 합성키 재설계. *NoSQL 원칙 학습 적용.*
**대사 마무리**: "ADR 15건으로 모든 결정의 근거·대안·번복을 기록했습니다." → 질의응답 준비도 어필.
**슬라이드**: 6p ADR before→after 4행 표.

### [6] 프로세스: WBS·개발환경·빌드배포·테스트·문서 — 0:50  (그룹 C)
**동시 득점(약 9항목)**:
- Cat1 `WBS`·`프로젝트 진행/완료`
- Cat3 `개발환경 구성`·`빌드/배포 개념`·`코드품질`·`단위 테스트`·`통합 테스트`·`GitHub 설치가이드`
- Cat2 `개발환경`·`빌드/배포 단계`
**말할 핵심**
- **WBS·진행**: 12주차 환경구축 → 13주차 메인기능 → 14주차 서브 → 15주차 최종. **GitHub Pages에서 실시간 확인** (← 잔디 15점 링크 연결).
- **개발환경**: Flutter SDK + Android 타깃, AWS SAM, CloudShell.
- **빌드/배포 단계**: 코드 → SAM 빌드 → **CloudShell `sam deploy`** (Access Key 미발급이라 콘솔 배포, ADR-010/013) → Lambda 6종 갱신. *"빌드와 배포 차이"를 한 문장으로 설명.*
- **테스트**: fn-authorizer **단위테스트 8 passed**, 배포 후 실환경 **통합(E2E) 검증**.
- **문서/설치가이드**: README + setup/deploy/testing 문서 + AGENTS.md (← 문서 5점), GitHub 클론·실행 가이드.
**슬라이드**: 7p WBS 간트 + 배포 파이프라인 + 테스트/문서 뱃지.

### [7] 활용·향후·나만의 기법 — 0:30  (그룹 F)
**동시 득점**: Cat1 `어떻게 활용할 것인지` + 가산점
**말할 핵심**
- 활용: "여행 전체 기록이 한 앱에 — 추천부터 정산까지." 향후: 실시간 통역(Transcribe) 확장 여지.
- **가산점(짧게)**: "에이전트·스킬·룰을 **단일 md(CLAUDE.md)로 통합**해 운영했고(+2), AI 에이전트 워크플로우를 적극 활용했으며(+1), 최신 LLM 기반 **암묵지 메모리**로 결정을 축적(+1)." → *길게 설명 금지(채점 지침).*
**슬라이드**: 8p 향후 로드맵 + 기법 1줄.

## 4. 질의응답(Cat2) 백업 — 답변 3종 암기 (v1 comment.md 부록 재활용)
1. **왜 Cognito가 아닌 소셜 OAuth?** 관리자 IAM에서 Cognito 차단 → Google ID 토큰을 `fn-authorizer`가 tokeninfo로 검증하는 무상태 방식 우회 (ADR-007).
2. **Bedrock 리전?** 인프라는 서울, Bedrock만 us-east-1 cross-region(~200ms). 프롬프트 최적화로 상쇄 (ADR-009).
3. **왜 DynamoDB(NoSQL)?** 여행 단위 단순 액세스 + On-Demand 과금이 PoC에 적합. `trip_id`+시각 SK 설계로 정렬 비용 0 (ADR-002/014/015).
- (보너스) **월 비용?** 무료티어 ~$2~3, 종료 후 $10~15.
