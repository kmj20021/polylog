### R-11: AI 의존 — 본인이 코드를 이해하지 못하는 위험

| 항목 | 내용 |
|---|---|
| **위험** | AI 코드 생성 도구(Claude, Copilot 등)에 과도하게 의존하여, 생성된 코드의 동작 원리를 개발자 본인이 설명하지 못함. 디버깅 시 원인 파악 불가, 발표 질의응답에서 "AI가 작성했다"는 답변만 가능하여 학습 목표 미달 판정, 사소한 수정에도 AI 없이는 진행 불가한 상태에 빠짐 |
| **발생 확률** | 상 |
| **영향도** | 상 — 디버깅 능력 상실 + 발표 평가 직접 영향 + 학습 목표(L-1~L-5) 전면 미달 |
| **해당 주차** | 10~15주차 전체 (특히 11~13주차 기능 집중 개발 시) |
| **상세 위험 시나리오** | **시나리오 A — 디버깅 불능**: fn-place Lambda에서 Rekognition 응답을 파싱하는 코드가 AI가 작성했는데, 특정 이미지에서 에러 발생 시 응답 구조를 이해하지 못해 수정 불가 → 해당 기능 개발 정체 / **시나리오 B — 발표 질의응답 실패**: "Step Functions의 Map State에서 병렬 처리 에러 핸들링을 어떻게 구현했나요?"라는 질문에 답변 불가 → 학습 프로젝트 평가 하락 / **시나리오 C — 연쇄 의존**: AI가 작성한 코드 위에 AI가 또 코드를 작성 → 전체 코드베이스를 이해하지 못하는 "블랙박스 누적" 발생 → 통합 테스트 시 원인 불명 버그 다발 |
| **대응 방안** | 아래 5단계 규칙을 엄격히 적용 |
| **트리거** | 특정 Lambda 함수의 동작 흐름을 3분 이내에 구두로 설명하지 못하면, 해당 함수를 AI 없이 처음부터 다시 작성 |

**AI 의존 방지 5단계 규칙**

```
┌─────────────────────────────────────────────────────────────────┐
│  1단계: 의사 코드 먼저 작성                                      │
│  ─ 구현할 로직을 한국어 의사 코드로 직접 작성                      │
│  ─ 입력 → 처리 단계 → 출력을 명확히 정의                          │
│                          ↓                                      │
│  2단계: AI로 구현 코드 생성                                      │
│  ─ 의사 코드를 기반으로 AI에게 코드 생성 요청                      │
│  ─ "이 로직을 Kotlin/Python으로 구현해줘" 형태                    │
│                          ↓                                      │
│  3단계: 대조 검증                                                │
│  ─ 생성 코드를 의사 코드와 한 줄씩 대조                           │
│  ─ 의사 코드에 없는 로직이 추가되었으면 이유를 반드시 이해          │
│                          ↓                                      │
│  4단계: "설명 테스트" 통과                                       │
│  ─ 해당 코드의 동작을 주석 없이 3분 이내에 구두 설명               │
│  ─ 실패 시 → 코드를 AI 없이 직접 다시 작성                       │
│                          ↓                                      │
│  5단계: 커밋 허용                                                │
│  ─ 위 4단계를 모두 통과한 코드만 커밋                             │
│  ─ 블랙박스 코드는 절대 커밋하지 않음                             │
└─────────────────────────────────────────────────────────────────┘
```

**AI 활용 허용/금지 영역**

| 구분 | 허용 | 금지 |
|---|---|---|
| **코드 생성** | 보일러플레이트, 반복 패턴, 단순 CRUD | 핵심 비즈니스 로직의 전체 위임 |
| **프롬프트** | 프롬프트 초안 검토·개선 제안 | Bedrock 프롬프트 전체를 AI에게 작성 위임 |
| **IaC** | SAM 문법 참고, 에러 메시지 해석 | template.yaml 전체를 AI에게 생성 위임 |
| **디버깅** | 에러 메시지 해석, 원인 후보 제시 | AI에게 "이 에러 고쳐줘"로 통째로 위임 |
| **학습** | 개념 설명, 예제 코드 비교 | 이해 없이 AI 답변을 복사-붙여넣기 |
