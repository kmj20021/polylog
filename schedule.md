### R-9: AI 의존 — 본인이 코드를 이해하지 못하는 위험

| 항목 | 내용 |
|---|---|
| **위험** | AI 코드 생성 도구에 과도하게 의존하여 생성된 코드의 동작 원리를 설명하지 못함. 디버깅 불가, 발표 질의응답 실패 |
| **발생 확률** | 상 |
| **영향도** | 상 — 디버깅 능력 상실 + 발표 평가 직접 영향 |
| **해당 주차** | 12~15주차 전체 |
| **대응 방안** | 아래 규칙을 엄격히 적용 |
| **트리거** | 특정 Lambda 함수의 동작 흐름을 3분 이내에 구두로 설명하지 못하면, 해당 함수를 AI 없이 처음부터 다시 작성 |

**AI 의존 방지 규칙**

```
1단계: 의사 코드 먼저 작성 (입력 → 처리 → 출력 정의)
    ↓
2단계: AI로 구현 코드 생성 (의사 코드 기반)
    ↓
3단계: 대조 검증 (생성 코드 ↔ 의사 코드 한 줄씩 비교)
    ↓
4단계: "설명 테스트" 통과 (3분 이내 구두 설명)
    ↓
5단계: 커밋 허용 (블랙박스 코드는 절대 커밋 금지)
```

| 구분 | 허용 | 금지 |
|---|---|---|
| **코드 생성** | 보일러플레이트, 반복 패턴, 단순 CRUD | 핵심 비즈니스 로직 전체 위임 |
| **프롬프트** | 프롬프트 초안 검토·개선 제안 | Bedrock 프롬프트 전체를 AI에게 작성 위임 |
| **IaC** | SAM 문법 참고, 에러 메시지 해석 | template.yaml 전체를 AI에게 생성 위임 |
| **디버깅** | 에러 메시지 해석, 원인 후보 제시 | "이 에러 고쳐줘"로 통째로 위임 |
